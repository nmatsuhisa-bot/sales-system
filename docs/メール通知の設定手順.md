# 承認依頼メールの設定手順（Gmail）

見積の承認依頼を、検印承認者にメールで通知する機能の設定方法です。
**設定するまでメールは送信されません**（承認依頼自体は成立し、画面の
「承認待ち」バナーで通知されます）。

---

## 1. Googleアプリパスワードを発行する

Gmailの通常のログインパスワードでは送信できません（Googleが拒否します）。
「アプリパスワード」という専用の16桁のパスワードを発行してください。

1. 送信に使うGoogleアカウントで https://myaccount.google.com/security を開く
2. **2段階認証プロセス** を有効にする（未設定の場合。アプリパスワードの発行に必須）
3. 同じセキュリティ画面で **アプリ パスワード** を開く
   （見つからない場合は https://myaccount.google.com/apppasswords を直接開く）
4. アプリ名に「販売管理システム」などと入力して作成
5. 表示される **16桁のパスワード** を控える（この画面を閉じると二度と表示されません）

> ⚠️ このパスワードはメール送信専用ですが、**漏れるとそのアカウントからメールを
> 送信できてしまいます**。取り扱いにご注意ください。不要になったら上記画面から失効できます。

---

## 2. Renderに環境変数を登録する

1. https://dashboard.render.com で **sales-backend** サービスを開く
2. 左メニューの **Environment** を開く
3. 以下を **Add Environment Variable** で追加する

| キー | 値 | 必須 |
|---|---|---|
| `MAIL_FROM` | 送信元のGmailアドレス（例: `xxx@gmail.com`） | ✅ |
| `MAIL_APP_PASSWORD` | 手順1で発行した16桁のアプリパスワード（空白は詰めてよい） | ✅ |
| `MAIL_FROM_NAME` | 差出人の表示名（未設定なら「井上電設 販売管理システム」） | |
| `APP_BASE_URL` | 画面のURL（未設定なら `https://sales-frontend-ybzn.onrender.com`） | |
| `API_BASE_URL` | APIのURL（未設定なら `https://sales-backend-7nzg.onrender.com`） | |

4. **Save Changes** を押す（サービスが自動で再起動します）

> パスワードはRenderの環境変数として保存され、画面やソースコードには出ません。

---

## 3. 承認者にメールアドレスを登録する

メールは **ユーザーマスタに登録されたメールアドレス** 宛に送られます。
ユーザー管理画面で、検印承認者それぞれのメールアドレスをご確認ください。

現在のテストユーザーは以下のとおりです（すべて matsuhisa27@gmail.com に届きます）。

| 氏名 | メールアドレス |
|---|---|
| 柴田 忠春 | matsuhisa27+shibata@gmail.com |
| 江里口 一博 | matsuhisa27+eriguchi@gmail.com |
| 井上 嗣夫 | matsuhisa27+inoue@gmail.com |

---

## 4. 動作確認

1. 見積を開き、検印に承認者を選んで **「承認依頼を送る」** を押す
2. 画面に「◯◯ に承認依頼しました（〜にメール送信）」と出れば送信成功
3. 受信メールを確認する

送信できない場合は、画面のメッセージに理由が表示されます。

| メッセージ | 対処 |
|---|---|
| メール未設定 | 手順2の環境変数が未登録。Renderの設定を確認 |
| ◯◯ のメールアドレスが未登録です | ユーザー管理でその人のメールアドレスを登録 |
| 送信失敗: SMTPAuthenticationError | アプリパスワードが誤っている。手順1をやり直す |

---

## 5. メールの内容

- **件名**: 【承認依頼】見積番号 件名（金額）
- **本文**: 見積番号・件名・注文主・金額・依頼者
- **添付**: 見積書 と 社内工数試算
- **承認リンク**: 開くだけで承認が完了します

### 承認リンクの制限

| 項目 | 仕様 |
|---|---|
| 有効期限 | **72時間**（`APPROVAL_LINK_HOURS`） |
| 使用回数 | **1回のみ**（使用後は「使用済み」と表示） |
| 再依頼時 | 以前の未使用リンクは失効する |
| 見積を編集した場合 | 承認が解除されるため、古いリンクは無効になる |
| 記録 | 使用日時と接続元IPを保存 |

> ⚠️ **リンクを開いた人が承認できる仕組みです。** メールを転送されると
> 第三者でも承認できてしまいます。より厳密な運用が必要な場合は、
> メール本文のもう一方のリンク（ログインしてから承認）をお使いください。

## 補足: 添付ファイルの形式について

現在、Renderのサーバに**PDF生成に必要なシステムライブラリが入っていない**ため、
添付は **HTMLファイル** になります（開くとブラウザで表示され、印刷からPDF化できます）。

PDFで添付するには、Renderのサービス設定を **Docker** に変更する必要があります
（`backend/Dockerfile` に必要なライブラリと日本語フォントを追加済み）。

| 方式 | 添付形式 | 対応 |
|---|---|---|
| 現在（ネイティブPythonビルド） | HTML | 追加作業なし |
| Dockerに変更 | **PDF** | Renderのダッシュボードで Runtime を Docker に変更 |
