# 承認依頼メールの設定手順（Google Workspace）

見積の承認依頼を、検印承認者にメールで通知する機能の設定方法です。
**設定するまでメールは送信されません**（承認依頼自体は成立し、画面の
「承認待ち」バナーで通知されます）。

送信方式は **SMTP（smtp.gmail.com:587）＋ アプリパスワード** です。
Google Workspace のアカウントでも、個人のGmailと同じ手順で送信できます。

---

## 0. 送信サービスの選択肢

実装は **汎用SMTP** のため、Google Workspace 以外でも
**環境変数を変えるだけで切り替えられます**（コード修正不要）。

| サービス | 無料枠 | 独自ドメイン送信 | 備考 |
|---|---|---|---|
| **Google Workspace**（現行） | 契約に含まれる | 設定済み（DNS作業不要） | 管理者がアプリパスワードを禁止していると使えない |
| **Resend** | 3,000通/月・100通/日（永続） | SPF/DKIMの登録が必要 | 開発者向け。設定が簡単 |
| **Brevo** | 300通/日・9,000通/月（永続） | SPF/DKIMの登録が必要 | 無料枠が最も大きい |
| **Amazon SES** | 実質従量（1,000通あたり約$0.10） | SPF/DKIMの登録が必要 | 最安。ただし**サンドボックス解除の申請**が必要 |
| **SendGrid** | **無料枠は2025年5月に廃止** | SPF/DKIMの登録が必要 | 60日試用後は $19.95/月〜 |
| Microsoft 365 | 契約に含まれる | 設定済み | M365をお使いの場合の選択肢 |

### 設定例（サービス別）

| 環境変数 | Google Workspace | Resend | Brevo | SendGrid |
|---|---|---|---|---|
| `SMTP_HOST` | （既定） | `smtp.resend.com` | `smtp-relay.brevo.com` | `smtp.sendgrid.net` |
| `SMTP_USER` | （不要） | `resend` | 登録メールアドレス | `apikey` |
| `MAIL_FROM` | 送信元アドレス | 送信元アドレス | 送信元アドレス | 送信元アドレス |
| `MAIL_APP_PASSWORD` | アプリパスワード | APIキー | SMTPキー | APIキー |

> ⚠️ **本件の送信量はごく少量**（承認依頼のみ・1日数通）です。どのサービスでも
> 無料枠に十分収まります。選定は「料金」より「**設定の手間**」で判断してください。

### 判断のめやす

- **管理者がアプリパスワードを許可できる** → **Google Workspace のまま**が最も簡単です。
  DNS（SPF/DKIM）の設定が既に済んでおり、追加作業がありません
- **許可が下りない／送信専用にしたい** → **Resend** か **Brevo** をお勧めします。
  ただし `@inoue-d.co.jp` で送るには、**ドメインのDNSにSPF/DKIMレコードを追加**する必要があります
  （これを行わないと迷惑メール判定されやすくなります）

## 1. Googleアプリパスワードを発行する

通常のログインパスワードでは送信できません（Googleが拒否します）。
「アプリパスワード」という専用の16桁のパスワードを発行してください。

1. 送信に使うアカウントで https://myaccount.google.com/security を開く
2. **2段階認証プロセス** を有効にする（未設定の場合。アプリパスワードの発行に必須）
3. https://myaccount.google.com/apppasswords を開く
4. アプリ名に「販売管理システム」などと入力して作成
5. 表示される **16桁のパスワード** を控える（この画面を閉じると二度と表示されません）

### ⚠️ Google Workspace 固有の注意

アプリパスワードは **管理者が組織全体で禁止できる** 設定項目です。
手順3の画面が開けない・「利用できません」と出る場合は、管理者による制限が原因です。

管理コンソール（admin.google.com）で次を確認してください。

| 場所 | 設定 |
|---|---|
| セキュリティ → 認証 → **2段階認証プロセス** | 対象ユーザーで有効にできること |
| セキュリティ → アクセスとデータ管理 → **安全性の低いアプリ / アプリパスワード** | アプリパスワードの発行を許可 |

**送信専用のアカウントを1つ用意する**ことをお勧めします（例: `system@…`）。
個人アカウントのアプリパスワードを使うと、退職・パスワード変更で送信が止まります。

> ⚠️ このパスワードはメール送信専用ですが、**漏れるとそのアカウントからメールを
> 送信できてしまいます**。取り扱いにご注意ください。不要になったら上記画面から失効できます。

### 参考: SMTPリレーを使う場合

Workspaceには「SMTPリレー」という別方式もありますが、**送信元IPの許可登録が必要**で、
Renderは送信元IPが固定されないため実用的ではありません。アプリパスワード方式を推奨します。

## 2. Renderに環境変数を登録する

1. https://dashboard.render.com で **sales-backend** サービスを開く
2. 左メニューの **Environment** を開く
3. 以下を **Add Environment Variable** で追加する

| キー | 値 | 必須 |
|---|---|---|
| `MAIL_FROM` | 送信元のGmailアドレス（例: `xxx@gmail.com`） | ✅ |
| `MAIL_APP_PASSWORD` | 手順1で発行した16桁のアプリパスワード（空白は詰めてよい） | ✅ |
| `MAIL_FROM_NAME` | 差出人の表示名（未設定なら「井上電設 販売管理システム」） | |
| `APP_BASE_URL` | 画面のURL（未設定なら `https://sales-frontend-ybzn.onrender.com`） | |
| `API_BASE_URL` | APIのURL（未設定なら `https://sales-backend-7nzg.onrender.com`） | |

4. **Save Changes** を押す（サービスが自動で再起動します）

> パスワードはRenderの環境変数として保存され、画面やソースコードには出ません。

## 3. 承認者にメールアドレスを登録する

メールは **ユーザーマスタに登録されたメールアドレス** 宛に送られます。
ユーザー管理画面で、検印承認者それぞれのメールアドレスをご確認ください。

現在のテストユーザーは以下のとおりです（すべて matsuhisa27@gmail.com に届きます）。

| 氏名 | メールアドレス |
|---|---|
| 柴田 忠春 | matsuhisa27+shibata@gmail.com |
| 江里口 一博 | matsuhisa27+eriguchi@gmail.com |
| 井上 嗣夫 | matsuhisa27+inoue@gmail.com |

## 4. 動作確認

1. 見積を開き、検印に承認者を選んで **「承認依頼を送る」** を押す
2. 画面に「◯◯ に承認依頼しました（〜にメール送信）」と出れば送信成功
3. 受信メールを確認する

送信できない場合は、画面のメッセージに理由が表示されます。

| メッセージ | 対処 |
|---|---|
| メール未設定 | 手順2の環境変数が未登録。Renderの設定を確認 |
| ◯◯ のメールアドレスが未登録です | ユーザー管理でその人のメールアドレスを登録 |
| 送信失敗: SMTPAuthenticationError | アプリパスワードが誤っている。手順1をやり直す |

---

## 5. メールの内容

- **件名**: 【承認依頼】見積番号 件名（金額）
- **本文**: 見積番号・件名・注文主・金額・依頼者
- **添付**: 見積書 と 社内工数試算
- **承認リンク**: 開くだけで承認が完了します

### 承認リンクの制限

| 項目 | 仕様 |
|---|---|
| 有効期限 | **72時間**（`APPROVAL_LINK_HOURS`） |
| 使用回数 | **1回のみ**（使用後は「使用済み」と表示） |
| 再依頼時 | 以前の未使用リンクは失効する |
| 見積を編集した場合 | 承認が解除されるため、古いリンクは無効になる |
| 記録 | 使用日時と接続元IPを保存 |

> ⚠️ **リンクを開いた人が承認できる仕組みです。** メールを転送されると
> 第三者でも承認できてしまいます。より厳密な運用が必要な場合は、
> メール本文のもう一方のリンク（ログインしてから承認）をお使いください。

## 補足: 添付ファイルの形式について

添付は **PDF** です。Renderの設定変更は不要です。

当初はWeasyPrintを使いましたが、pango/cairo等のシステムライブラリが必要で、
Renderのネイティブビルド（`pip install` のみ・`apt` 不可）では動きませんでした。
現在は **xhtml2pdf（純Python）** を使っており、追加のシステムライブラリは要りません。
日本語は reportlab 同梱のCIDフォントで描画するため、フォントファイルの配置も不要です。

万一PDFを生成できない環境になった場合は、**自動的にHTML添付へ切り替わります**
（開くとブラウザで表示され、印刷からPDF化できます）。メールが送れなくなることはありません。

> 帳票のレイアウトを変更したときは、PDFの見た目も確認してください。
> xhtml2pdfはCSSの対応範囲が狭く、flexboxなどは反映されません（帳票は表組み中心のため通常は問題ありません）。
