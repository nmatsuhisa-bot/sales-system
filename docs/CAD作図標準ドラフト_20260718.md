# CAD作図標準（ドラフト）— 見積自動化・AI学習のための作図規約

作成日: 2026-07-18
位置づけ: 会議(2026-07-17)の宿題「立体配置図の作成標準を整理して文書化する（期限 7/31）」の叩き台。
実図面6件（本多木工所・西北プライウッド・川井林業・新栄合板・釧路ウッドプロダクツA/B）の
機械解析（LibreDWG + ezdxf）で判明した事実に基づく。

## 0. この規約の目的

図面から見積を自動生成するには、コンピュータが図面の中身を「識別」できる必要がある。
現状の図面で機械的に読み取れるもの・読み取れないものは次のとおり（実測）。

| 情報 | 現状 | 原因 |
|---|---|---|
| 機器の型式・台数 | ✅ 読める | ブロック名に型式が入っている |
| モータ容量・処理風量 | ✅ 読める | テキストとして記載されている |
| ダクト径 | ✅ 読める | φ表記がテキストにある |
| **ダクトの長さ・本数** | ❌ **読めない** | 輪郭2本の平行線で描かれ、建屋の線と区別できない |
| 自社/他社製品の区別 | 🔶 半分読める | レイヤー分離は存在するが名称・運用が図面ごとに揺れる |
| 高さ方向（立ち上がり） | 🔶 半分読める | `(6000H)` 等の文字注記のみ。記載は任意 |

本規約は「❌・🔶を✅に変える」ための最小限のルールである。

## 1. ブロック名の規約

### 1.1 自社製品ブロックの命名

```
<型式>_<ビュー>
例: BFR5X6_平面 / SCA675_正面 / RV30X80_側面
```

- **型式は英数字のみ・区切りは `X`**（`×` や全角は使わない）: `BFR5X6` `RV30X80` `PLD7.5`
- ビュー名は次の語に限定: `平面` `正面` `側面L` `側面R` `裏面` `立面` `基礎` `略図`
- **工番をブロック名に入れない**（禁止例: `240001A-SCA675平面`）。
  工番が入ると図面を流用したときに前案件の工番が残り、誤読の原因になる（川井林業で実例）
- 状態修飾（`単色` `消火配管_` 等）は型式の**後ろ**に付ける: `SCA675_消火配管_平面`

### 1.2 現状の揺れの実例（禁止パターン集）

実図面から採取した揺れ。同じ意味がこれだけの表記に分かれており、機械照合を妨げている。

| 現状の実例 | 問題 |
|---|---|
| `BFQ7-60平面` | 区切りが `-`、`-60` の意味が不明 |
| `単色PL200K209側面` | 接頭辞 `単色`、`K209` の意味が不明 |
| `240001A-SCA675平面` | 工番入り（流用時に残留する） |
| `260001A_SCA844_本体略図` | 別案件の工番が残留 |
| `BFR5W6LBLOCK024` | 自動生成名のまま |
| `BFR3×5_F_正面` と `BFR3X5_F_側面` | `×` と `X` の混在 |

### 1.3 サフィックスコードの定義書を作る

`-60` `K209` `-2R52` 等の既存コードは意味の定義書（1枚）を作成し、
新規図面では **型式＋定義済みコードのみ**を使う。

## 2. レイヤーの規約

### 2.1 固定レイヤー構成

| レイヤー名 | 用途 | 備考 |
|---|---|---|
| `INOUE_機器` | 自社製品のブロック配置 | 見積対象 |
| `INOUE_ダクト` | **ダクト中心線（後述3章）** | 見積対象・長さ積算に使う |
| `INOUE_ダクト外形` | ダクトの見た目（輪郭・ハッチ） | 印刷用。積算には使わない |
| `背景_建屋` | 建屋・構造 | 客先資産 |
| `背景_既設` | 客先の既設機器・他社製品 | 見積対象外の判定に使う |
| `注記` | 文字・引出線 | |
| `寸法` | 寸法 | `Defpoints` に作図しない |

- 現状の `製作図`/`制作図`（同音異字が混在）、`LEVEL001`〜、`背景１`〜 は新規図面では使わない
- **`Defpoints` レイヤーに実体を描かない**（新栄合板で幾何の大半が Defpoints にあり、
  非表示・非印刷の特殊レイヤーのため解析も印刷も阻害している）

### 2.2 自社/他社の区別

会議 3.2 の「レイヤーや色分けで区別」は、上記 `INOUE_*` / `背景_*` の接頭辞で実現する。
色ではなく**レイヤー名で判定する**（色は印刷時に失われるため）。

## 3. ダクトの作図規約 【最重要】

**ダクトは `INOUE_ダクト` レイヤーに、系統ごとのポリライン（中心線1本）で描く。**

- 1系統 = 1ポリライン（分岐点で分割）
- ポリラインの**線幅または拡張データに径を持たせる**か、少なくとも系統ごとに
  `φ350` の注記を中心線に沿えて置く
- 立ち上がり（垂直部）は平面図では長さが出ないため、**`(6000H)` 形式の注記を必須**とする
  （現状も西北プライウッドで `φ300SD(6000H)` の記載例あり。これを標準化する）
- 見た目用の輪郭線・ハッチは `INOUE_ダクト外形` に分けて描く

このルールだけで「中心線の総延長 × 系統別の径」が機械集計できるようになり、
ダクト部品の見積が図面から自動算出可能になる。
（検証済み: 図面の単位系は mm（$INSUNITS=4）で、座標から実寸が正しく計算できる）

### 3.1 現行の暫定運用（ダクト概算） ★係数は仮値・要確認

作図標準が導入されるまでの間、システムは次の暫定式でダクトを**概算**する
（`backend/app/cad_extract.py`）。

```
概算金額 = Σ 径ごと（ 径(mm) × 20円 × 想定延長(m) ）
想定延長 = 図面の径注記(φ表記)の件数 × 6m
```

| パラメータ | 暫定値 | 意味 |
|---|---|---|
| `DUCT_RATE_PER_MM_M` | 20 円/(mm・m) | φ375なら 7,500円/m |
| `DUCT_DEFAULT_RUN_M` | 6 m | 径注記1件あたりの想定配管長 |

**原本見積との突合（概算の妥当性確認）**

| 案件 | 原本のダクト部品（売価） | 本式の概算 | 差 |
|---|---|---|---|
| 新栄合板 260024A | 550,000 | 467,000 | −15% |
| 西北プライウッド 250365A | 500,000 | 623,000 | +25% |

桁は合っているが±25%の幅がある。**受注判断に使える精度ではない**ため、
生成された見積では必ず担当者が確認・修正すること。
3章の中心線ルールが導入されれば、この暫定式は実長ベースの積算に置き換える。

## 4. テキスト注記の規約

機器仕様は現状でも図面内テキストから読み取れている（例: 新栄合板
`バグフィルター集塵機 / BFQ10V / 集塵ファン:7.5kw / シェーカー:0.2kw`）。
これを必須項目として標準化する。

- 主要機器の脇に**仕様ブロック**（MTEXT 1個）を置く。記載順:
  `品名 / 型式 / 処理風量 / モータ容量`
- 既設・支給品には **`【既設】`** を先頭に付ける（川井林業の `【既設使用】` 記載を標準化。
  見積対象外の自動判定に使う）
- 制御盤は `制御盤 / 電気容量:○kw / 電源:AC200V` の形式で記載

## 5. ファイル・リビジョンの規約

- ファイル名は `<工番>_<顧客名>_<棟・系統>.dwg`（現状踏襲）。
  **ただし工番の正はファイル名ではなく図面表題欄**とする
- **DWG保存と同時にDXF（AutoCAD 2018 DXF形式）も書き出す**。
  理由: 解析パイプラインの入力。AutoCADなら `DXFOUT` 1コマンド。
  （LibreDWGでR2018 DWGの読み取りは概ね可能だが、釧路製材棟280001A=101MBで
  読み取り失敗が発生しており、DXF併存が確実）
- 図面を改訂して見積と数値が変わる場合（処理風量等）は**見積番号の枝番と図面リビジョンを対応**させる
  （川井林業で図面 550m3/min vs 見積 500㎥/min の不一致が実在し、どちらが最新か判別不能）

## 6. AI学習データ提出時の付帯情報（8/15期限のデータセット用）

CAD＋見積のペアを提出する際、次の3点を1行ずつ添える:

1. 図面と見積のリビジョン対応（どの図面版がどの見積版に対応するか）
2. 図面内の既設機器の有無（`【既設】` 注記が無い旧図面の場合）
3. 受注の成否と最終金額（値引き後）

## 付録: 本規約の効果（実測に基づく見込み）

| 見積の構成 | 現状の図面 | 本規約適用後 |
|---|---|---|
| 大項目1〜5の骨格 | ✅ 自動生成可（プロトタイプ検証済み） | ✅ 精度向上 |
| 機器本体の型式・台数 | ✅ 取得可 | ✅ 表記統一で照合精度向上 |
| 機器本体の価格 | 🔶 パターンマスタ照合（BFQ/BFR実装済） | ✅ |
| ダクト部品（径・長さ・金額） | ❌ 径のみ | ✅ **中心線ルールで自動積算可** |
| 取付工費・運送費 | ❌ 図面外 | ❌ 図面外（機器台数からの概算は将来課題） |

会議 3.1 の「立体配置図があれば約95%の精度」に対し、本規約は平面図＋高さ注記で
ダクト積算まで到達する現実的な中間解である。
